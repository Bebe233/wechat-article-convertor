# 事实表 — Markdown转微信（示例项目）

> 本目录为虚构演示，便于对照 pipeline 产物格式。非真实商业产品。

## 产品一句话

面向内容创作者的开源小工具，把 Markdown 草稿一键整理成可粘贴到微信公众号编辑器的 HTML 片段。

## 功能列表

- 读取本地 `.md` 文件，按章节拆分为段落与标题
- 内置与设计规范一致的 HTML snippet 模板（H2/H3/引用/列表/高亮/CTA）
- 输出配图占位注释，提醒在公众号后台上传图片
- 示例目录 `articles/_example/` 展示 facts → raw → wechat 全流程

## 技术亮点

- 纯 Cursor Skill 编排，无 npm 运行时依赖（v1）
- 样式全部内联，适配公众号粘贴限制
- 双卡点（facts、raw）减少返工

## 差异化

- 不绑定第三方排版 SaaS；规范与片段在仓库内版本化管理
- 营销成稿与微信 HTML 分离，便于改稿而不动样式

## 目标用户

- 需要定期发公众号的技术团队、独立开发者、运营同学
- 已有 Markdown 博客/文档，希望复用到微信渠道

## 可引用数据/案例

- 示例工作流可在 1 个会话内完成调研到排版（取决于仓库大小）
- 本仓库 `articles/_example/` 即标准样例

## 不宜夸大项

- v1 不提供一键群发、阅读量统计
- 不保证所有第三方 Markdown 扩展语法都能映射
