# 小红书笔记格式规范

本仓库的小红书笔记以**纯文本 Markdown** 实现，便于复制到小红书创作者中心。排版 Skill 不得偏离下列约束。微信排版见 [design-system.md](design-system.md)。

## 字数与结构

| 字段 | 限制 | 说明 |
|------|------|------|
| 发布标题 | ≤20 字 | 抓眼球、可含 emoji；不含 `#` 话题 |
| 正文 | ≤1000 字 | 短段为主，2–4 行一段；列表感强；轻强调用「」而非 `**` |
| 话题标签 | 5–10 个 | 格式 `#标签`，空格分隔；须与 facts 相关 |

## 语气类型

| 类型 | 适用 | 特征 |
|------|------|------|
| 种草 | 产品推广、工具安利 | 第一人称、体验感、轻 emoji |
| 干货 | 教程、方法论 | 条理清晰、少夸张、步骤/要点列表 |
| 测评 | 对比、选型 | 客观陈述优缺点、数据引用须来自 facts |

语气由用户每次指定，不在规范中写死。

## 配图占位语法

成稿阶段（`raw-xhs.md`）使用 HTML 注释占位；终稿（`xhs.md`）转为配图清单。

### 封面（首张轮播图）

```html
<!-- COVER: {类型描述} | 比例 3:4 | 备注 {可选} -->
```

### 内页图

```html
<!-- IMAGE: {类型描述} | 比例 3:4 | 备注 {可选} -->
```

示例：

```html
<!-- COVER: 封面：笔记本电脑旁 Markdown 与手机并排，暖色办公 | 比例 3:4 | 备注 无 Logo -->
<!-- IMAGE: 流程图：调研→成稿→排版三步 | 比例 3:4 | 备注 简约扁平风 -->
```

### 配图数量与比例

- 默认 **4–6 张**竖图占位（1 封面 + 3–5 内页）
- 比例默认 **3:4**（小红书推荐竖图）
- **仅占位**：不写外链、不生成图片文件、不使用 `<img>`

## 成稿结构（raw-xhs.md）

```markdown
# 笔记标题（≤20字）

<!-- COVER: ... -->

（hook 短段）

<!-- IMAGE: ... -->

- 要点 …

## 话题标签
#标签1 #标签2
```

- 禁止微信 snippet 语义标记（如 `【highlight】`、`【cta】`）
- 禁止 HTML 标签与外链图片
- 禁止 Markdown 加粗（`**`）；轻强调用中文直角引号「」包裹

## 终稿结构（xhs.md）

由 **xhs-format** Skill 从已确认的 `raw-xhs.md` 生成：

```markdown
## 发布标题
（≤20字，可直接复制）

## 正文
（纯文本分段，保留 emoji；轻强调用「」）

## 话题标签
#xxx #yyy

## 配图清单
1. [封面 3:4] ...
2. [内页 3:4] ...
```

## 发布步骤

1. 打开 `articles/<slug>/xhs.md`
2. 复制「发布标题」到小红书标题栏
3. 复制「正文」+「话题标签」到正文区（标签也可在发布页单独添加）
4. 按「配图清单」顺序上传竖图

## 与微信的差异

| 维度 | 微信 | 小红书 |
|------|------|--------|
| 成稿文件 | `raw-wechat.md` | `raw-xhs.md` |
| 终稿文件 | `wechat.html` | `xhs.md` |
| 排版 | snippets + 内联 style | 纯文本，无 HTML |
| 配图语法 | 仅 `IMAGE` | `COVER` + `IMAGE` |
| 默认比例 | 16:9 / 2.35:1 等 | 3:4 竖图 |
